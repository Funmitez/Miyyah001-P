    🔹 VPC Setup with Terraform

    Designed and deployed a secure AWS Virtual Private Cloud (VPC) using Terraform, following best practices for high availability. Configured public and private subnets, internet gateway, route tables, and NAT gateway to allow outbound internet access for private instances while restricting inbound traffic. Deployed EC2 instances into private subnets while keeping a bastion host in the public subnet for secure SSH access. 

    
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
  enable_dns_support = true
  enable_dns_hostnames = true
  tags = { Name = "main-vpc" }
}

resource "aws_subnet" "public" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.1.0/24"
  map_public_ip_on_launch = true
  availability_zone = "us-east-1a"
  tags = { Name = "public-subnet" }
}

    Key Learnings: Infrastructure as Code, secure networking design, automation with Terraform.

    🔹 Auto Scaling with Load Balancer

    Implemented an AWS Auto Scaling Group (ASG) with an Application Load Balancer (ALB) to automatically scale EC2 instances based on traffic demand. Configured health checks to replace unhealthy instances and maintain high availability. The ALB distributed traffic evenly while scaling policies automatically adjusted capacity during demand spikes. 

    
aws autoscaling create-auto-scaling-group \
  --auto-scaling-group-name myASG \
  --launch-configuration-name myLC \
  --min-size 2 \
  --max-size 5 \
  --desired-capacity 2 \
  --availability-zones us-east-1a us-east-1b

    Key Learnings: Fault tolerance, load balancing, cost-optimized scaling.

    🔹 CI/CD Pipeline with Jenkins

    Built an automated CI/CD pipeline using Jenkins integrated with GitHub. The pipeline performed code checkout, build, test, and deployment stages. Notifications were sent to Slack upon successful or failed builds. This automation reduced manual errors and accelerated software delivery cycles. 

    
pipeline {
  agent any
  stages {
    stage('Checkout') {
      steps { git 'https://github.com/username/repo.git' }
    }
    stage('Build') {
      steps { sh 'mvn clean install' }
    }
    stage('Test') {
      steps { sh 'mvn test' }
    }
    stage('Deploy') {
      steps { sh './deploy.sh' }
    }
  }
  post {
    success { slackSend channel: '#devops', message: 'Deployment Successful ✅' }
    failure { slackSend channel: '#devops', message: 'Build Failed ❌' }
  }
}

    Key Learnings: CI/CD automation, Jenkins pipelines, DevOps collaboration.

    🔹 Docker & Kubernetes Deployment

    Containerized a Node.js application using Docker and deployed it to a Kubernetes cluster. Used Helm charts for managing releases and scaling pods automatically based on resource usage. Kubernetes handled self-healing by replacing unhealthy pods and ensured service discovery with a LoadBalancer service. This ensured high availability and fault tolerance. 

    
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp-deployment
spec:
  replicas: 3
  selector:
    matchLabels: { app: myapp }
  template:
    metadata: { labels: { app: myapp } }
    spec:
      containers:
      - name: myapp
        image: my-dockerhub-user/myapp:latest
        ports: [{ containerPort: 3000 }]
---
apiVersion: v1
kind: Service
metadata:
  name: myapp-service
spec:
  type: LoadBalancer
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 3000

    Key Learnings: Containers, orchestration, Helm, and zero-downtime deployments.

    🔹 Serverless Application with AWS Lambda & API Gateway

    Designed and deployed a serverless application on AWS using Lambda, API Gateway, and DynamoDB. API Gateway exposed REST endpoints, which triggered Lambda functions to process data and store results in DynamoDB. This eliminated the need to manage servers and ensured scalability and cost-efficiency with a pay-per-execution model. 

    
import json, boto3
dynamodb = boto3.resource('dynamodb')
table = dynamodb.Table('Users')

def lambda_handler(event, context):
    user = json.loads(event['body'])
    table.put_item(Item=user)
    return { 'statusCode': 200, 'body': 'User saved successfully!' }

    Key Learnings: Serverless computing, event-driven design, AWS integrations.
